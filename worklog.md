# Work Log - Task 2

## Issue 1: Blank Page Fix

### Root Cause
The blank page was caused by a **stale Prisma Client** in the Next.js dev server cache. The Prisma schema had `priorityCourses` and `isAcceptingApplications` fields, and the database had the data, but the Turbopack-cached Prisma Client was from an older schema version that didn't include these fields. 

The SQL queries generated by the cached Prisma Client omitted these columns:
```sql
-- Missing: priorityCourses, isAcceptingApplications
SELECT id, name, provider, ... FROM Scholarship WHERE isActive = ?
```

This caused the API to return scholarship objects without `priorityCourses` and `isAcceptingApplications`, leading to missing data in the frontend components.

### Fix Applied
1. **Cleared the `.next` cache** (`rm -rf .next`) to force Turbopack to recompile with the updated Prisma Client
2. **Regenerated Prisma Client** with `bunx prisma generate` to ensure the latest schema is used
3. **Restarted the dev server** via the watchdog script to pick up the new Prisma Client
4. After restart, the SQL queries now include all fields:
```sql
SELECT id, name, ..., priorityCourses, ..., isAcceptingApplications, ... FROM Scholarship
```

### Additional Fix
- Fixed Footer hydration mismatch by replacing `{new Date().getFullYear()}` with a static `2026` to prevent server/client year differences
